# FlowGrid Memory Runtime Video V3 Design

## Purpose

Explain one synthetic lifecycle without turning a new request into silent
authority. The video is proof for the CockroachDB x AWS hackathon, not a
general FlowGrid product demo.

## Visual System

- Paper: `#f5eedb`
- Ink and confirmed structure: `#26221a`
- Pending revision: seal red `#8c2f23`
- Qualifying evidence: signal blue `#2a6df5`
- Superseded history: muted brown `#7a6244`
- Brand signature: the 人工智障研究所 seal appears only in the opening and closing.

## Scene Rules

1. Every state relation must have an exact visible attachment point.
2. Connection labels sit on a clear paper field, never float under an unrelated line.
3. Red means proposed or not-yet-authorized, not failure.
4. Blue means evidence applied, not confirmation.
5. Confirmation is a human-authorized state expressed by ink, hierarchy, and
   explicit labels rather than a decorative color block.
6. The final card names the real deployed pieces: synthetic data, AWS Lambda +
   API, CockroachDB lifecycle state, the read-only public demo, and the
   Institute source.

## Export Gate

The current visuals are ready for final export after the closing narration is
replaced. The old clip says the AWS deployment is planned; that wording is no
